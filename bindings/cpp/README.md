# unique (C++)

> One API surface, infinite languages. The C++ binding for the Unique.js framework.

## Install

The C++ binding is a header-only wrapper around the C ABI. You need:

1. The `unique.h` C header (generated by `cbindgen` from `core/`).
2. The `unique.hpp` C++ wrapper (in this directory).
3. The `unique_core` shared library (built with `--features ffi`).

```bash
# From the repo root:
cargo build -p unique-core --release --features ffi
# This generates core/unique.h and the libunique_core.so / .dylib / .dll.
```

## Quickstart

```cpp
#include "unique.hpp"

int main() {
    unique::UniqueRouter router;

    router.get("/hello", [](unique::Request& req, unique::Response& res) {
        res.status(200).json(R"({"message":"world"})");
    });

    unique::UniqueServer server(std::move(router));
    server.listen(3000);
}
```

Build:

```bash
clang++ -std=c++17 \
  -I/path/to/repo/core \
  -I/path/to/repo/bindings/cpp \
  -L/path/to/repo/target/release \
  -lunique_core \
  -o app main.cpp
```

## API reference

### `unique::UniqueRouter`

Construct a new router. Call `.get(path, handler)`, `.post(path, handler)`, etc.
to register routes. The handler is a `std::function<void(Request&, Response&)>`.

### `unique::UniqueServer`

Construct from a `UniqueRouter` (moved). Call `.listen(port)` to start.

### `unique::Request`

- `req.param("id")` — route parameter
- `req.header("content-type")` — request header
- `req.body()` — request body as `std::string`

### `unique::Response`

Chainable: `res.status(200).header("x-foo", "bar").json("...")`.

- `res.status(int)` — set status code
- `res.header(key, value)` — set header
- `res.json(string)` — set JSON body
- `res.text(string)` — set text body
- `res.html(string)` — set HTML body
- `res.error(code, message)` — set error response
