# kungfu (C++)

> One API surface, infinite languages. The C++ binding for the Kungfu.js framework.

## Install

The C++ binding is a header-only wrapper around the C ABI. You need:

1. The `kungfu.h` C header (generated by `cbindgen` from `core/`).
2. The `kungfu.hpp` C++ wrapper (in this directory).
3. The `kungfu_core` shared library (built with `--features ffi`).

```bash
# From the repo root:
cargo build -p kungfu-core --release --features ffi
# This generates core/kungfu.h and the libkungfu_core.so / .dylib / .dll.
```

## Quickstart

```cpp
#include "kungfu.hpp"

int main() {
    kungfu::KungfuRouter router;

    router.get("/hello", [](kungfu::Request& req, kungfu::Response& res) {
        res.status(200).json(R"({"message":"world"})");
    });

    kungfu::KungfuServer server(std::move(router));
    server.listen(3000);
}
```

Build:

```bash
clang++ -std=c++17 \
  -I/path/to/repo/core \
  -I/path/to/repo/bindings/cpp \
  -L/path/to/repo/target/release \
  -lkungfu_core \
  -o app main.cpp
```

## API reference

### `kungfu::KungfuRouter`

Construct a new router. Call `.get(path, handler)`, `.post(path, handler)`, etc.
to register routes. The handler is a `std::function<void(Request&, Response&)>`.

### `kungfu::KungfuServer`

Construct from a `KungfuRouter` (moved). Call `.listen(port)` to start.

### `kungfu::Request`

- `req.param("id")` — route parameter
- `req.header("content-type")` — request header
- `req.body()` — request body as `std::string`

### `kungfu::Response`

Chainable: `res.status(200).header("x-foo", "bar").json("...")`.

- `res.status(int)` — set status code
- `res.header(key, value)` — set header
- `res.json(string)` — set JSON body
- `res.text(string)` — set text body
- `res.html(string)` — set HTML body
- `res.error(code, message)` — set error response
